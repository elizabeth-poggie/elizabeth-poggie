# Question 1

What is the following in it’s non arrow syntax?

```js
let test = (x) => x + 2;
```

## Solution

```js
let test = function (x) {
  return x + 2;
};
```

# Question 3

Given the following code:

```js
let myArray = [
  { year: 2018, model: "X" },
  { year: 2016, model: "S" },
];

let test = myArray.map((car) => car.model + "-turbo");
```

Which statements are `true`?

- `myArray` will remain unchanged
- `myArray` will hold an array of strings: `["X-turbo", "S-turbo"]`
- `myArray` will hold an array with the same objects but with modified model names
- `test` will hold the value `undefined`
- `test` will hold an array with two values: `[undefined, undefined]`
- `test` will hold an array of strings: `["X-turbo", "S-turbo"]`

## Solution

- ✅ `myArray` will remain unchanged
- ❌ `myArray` will hold an array of strings: `["X-turbo", "S-turbo"]`
- ❌ `myArray` will hold an array with the same objects but with modified model names
- ❌ `test` will hold the value `undefined`
- ❌ `test` will hold an array with two values: `[undefined, undefined]`
- ✅ `test` will hold an array of strings: `["X-turbo", "S-turbo"]`

# Question 4

What is the datatype of each thing?

```js
let thing1 = new IntersectionObserver(thing2, thing3);
thing1.observe(thing4);
function thing2(thing5) {
  // console log the data-thingy
}
```

## Solution

- `thing1` 👉 `IntersectionObserver`. This used to observe changes in the intersection of a target element.
- `thing2` 👉 `Function`. This is the callback function.
- `thing3` 👉 `Object` or `undefined`. This represents the `options`
- `thing4` 👉 `Element`
- `thing5` 👉 `IntersectionObserverEntry`. It describes a single threshold change event for an "observed" element.

# Question 5

Fill in the for loop 👉

```js
function myFunction(entries) {
  entries.forEach((entry) => {
    // add code here
  });
}

let myObserver = new IntersectionObserver(myFunction);
myObserver.observe(myElement1);
myObserver.observe(myElement2);
```

When the function is called, set the class of any element that is intersecting to "seen"

## Solution

```js
if (entry.isIntersecting) {
  entry.target.classList.add("seen");
}
```

What's going on in this code?

- `entry.target` 👉 Refers to the actual DOM element being observed
- `classList.add("seen")` 👉 Adds the "seen" class to any element that is intersecting

# Question 6

What is the datatype of each thing?

```js
fetch(theURL)
  .then((result1) => {
    return result.json();
  })
  .then((result2) => {
    // do stuff lol
  })
  .catch((error) => {
    console.error("AHHHHH: ", error);
  });
```

## Solution

- `fetch(theURL)` 👉 `Promise<Response>`
- `result1` 👉 `Response`
- `result.json()` 👉 `Promise<Object>`
- `result2` 👉 `Object` (or `Array`, depending on the JSON)
- `(error) => console.error("AHHHHH: ", error)` 👉 `Function`
